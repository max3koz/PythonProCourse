1. Визначення термінів:
------------------------
XSS (Cross-Site Scripting): атака, що дозволяє вставити шкідливий JavaScript у сторінку.
SQL injection: атака, що дозволяє змінити SQL-запит через введення користувача.
CSRF (Cross-Site Request Forgery): атака, що змушує користувача виконати небажаний запит.
Clickjacking: обман користувача, щоб він натиснув на прихований елемент.
Параметризовані запити: SQL-запити з фіксованою структурою, що запобігають ін’єкціям.
Middleware: проміжний компонент Django, що обробляє запити та відповіді.
Salt: випадкове значення, що додається до пароля перед хешуванням.
Hashing: перетворення даних у незворотну хеш-форму (наприклад, SHA-256).

================================================================================

2. Основні вразливості Django-додатка
------------------------------------
XSS: уникати safe в шаблонах, використовувати автозакодування.
SQL injection: використовувати ORM Django.
CSRF: вмикати csrf_token у формах.
Clickjacking: додати X-Frame-Options: DENY.
Витік даних: обмежити доступ до конфіденційної інформації.

================================================================================

3. Захист конфіденційних даних
---------------------------------
Використання хешування паролів (PBKDF2, Argon2)
Збереження закритих даних у .env або settings.py через os.environ
Викристання шифрування з використанням Fernet або Django Encrypted Fields.

================================================================================

4. Аутентифікація та авторизація в Django
-------------------------------------------
Стандартна система auth: логін/пароль, групи, права
Token-based (DRF): для API, зручно для мобільних
OAuth/Social login: через Google, Facebook

Переваги: інтеграція, безпека.
Недоліки: складність налаштування для нестандартних сценаріїв

================================================================================

5. Безпека сесій
------------------
Потрібно вмикати SESSION_COOKIE_SECURE та SESSION_COOKIE_HTTPONLY.
Потрібно унікати збереження конфіденційних даних у сесії.

Загрози: крадіжка cookie, фіксація сесії
Захист: HTTPS, короткий час життя сесії, перевірка IP/User-Agent

================================================================================

6. Захист від DDoS
--------------------
Використання rate limiting (наприклад, django-ratelimit)
Встановлення reverse proxy (Cloudflare, nginx)
Кешування сторінки (django-cache, Redis)
Відстеження трафіку і логів.

================================================================================

7. Захист від CSRF
--------------------
Вмикати django.middleware.csrf.CsrfViewMiddleware
Додавати {% csrf_token %} у всі POST-форми
Перевіряти заголовки Referer і Origin
Для API використовувати токени або SameSite cookies

================================================================================
